--- permalink: /mostly-so-i-dont-forget-to-look-into-it/index.html layout: post title: Mostly so I don't forget to look into it... published: true comments: true categories: [] --- In f#, type inference suffers badly when calling methods on an instance of a class initiated outside of the function in question.

I wonder if quotations would allow a general function "xx" to create functions "xxmethodname" such that:

xx -> instance.methodname -> xxmethodname

xxmethodname instance [args] = instance.methodname [args]

and if so how the type inference of the compiler would cope with it.
